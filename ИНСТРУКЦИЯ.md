# Бот-консультант Mare Blanco

Что умеет:
- 📏 **Подобрать размер** — по росту, обхвату груди и талии бот выдаёт рекомендуемый размер из вашей сетки.
- 🧺 **Уход за вещью** — по артикулу бот выдаёт инструкцию (текст вы заранее пишете в Google Таблице).

---

## Что нужно подготовить (3 ключа)

| Что | Где взять | Куда вставить |
|---|---|---|
| `BOT_TOKEN` | @BotFather → /newbot | Railway → Variables |
| `GOOGLE_SHEET_ID` | URL вашей таблицы (между /d/ и /edit) | Railway → Variables |
| `GOOGLE_CREDENTIALS_JSON` | JSON-ключ сервисного аккаунта Google | Railway → Variables |

---

## Шаг 1. Бот в Telegram

✅ Уже сделано — у вас есть `BOT_TOKEN`.

## Шаг 2. Google Таблица

1. https://sheets.google.com → создайте таблицу *Mare Blanco — бот*.
2. **ID таблицы** — часть URL между `/d/` и `/edit`. Сохраните.
3. Внизу слева создайте один лист с названием `Каталог`.
4. В первой строке листа `Каталог` введите шапку (4 колонки):

   | Артикул | Название | Материал | Уход |
   |---|---|---|---|

## Шаг 3. Сервисный аккаунт Google

1. https://console.cloud.google.com → *New Project* → имя `mare-blanco-bot` → Create.
2. Найдите и включите **Google Sheets API**, потом **Google Drive API** (кнопка *Enable*).
3. *APIs & Services* → *Credentials* → *+ Create Credentials* → **Service account** → имя `bot` → Create → Done.
4. Откройте созданный аккаунт → вкладка **Keys** → *Add Key* → *Create new key* → **JSON** → файл скачается.
5. В этом JSON найдите `"client_email"` — скопируйте адрес.
6. В Google Таблице → **Поделиться** → вставьте email → права **Редактор**.
7. Откройте JSON в Блокноте, выделите всё (Ctrl+A, Ctrl+C) — пригодится в шаге 5.

## Шаг 4. GitHub

1. https://github.com → войдите/зарегистрируйтесь.
2. **+** в правом верху → *New repository* → имя `mare-blanco-bot` → **Private** → Create.
3. На странице репозитория → *uploading an existing file* → перетащите все файлы из `C:\Users\Admin\Documents\Claude\Projects\Боты\` **кроме**: `.env`, `qr-code.png`, скачанного JSON, папки `__pycache__`.
4. *Commit changes*.

## Шаг 5. Railway

1. https://railway.com → войдите через GitHub.
2. *New Project* → *Deploy from GitHub repo* → выберите `mare-blanco-bot`.
3. Первая сборка упадёт — это нормально, переменных ещё нет.
4. Сервис → **Variables** → добавьте 3 переменные:

   | Name | Value |
   |---|---|
   | `BOT_TOKEN` | ваш токен |
   | `GOOGLE_SHEET_ID` | ID таблицы (шаг 2.2) |
   | `GOOGLE_CREDENTIALS_JSON` | всё содержимое JSON (одной строкой) |

5. Railway передеплоит автоматически. *Deployments* → *View Logs* → должна появиться строка `Бот запущен. Жду сообщений…`.

## Шаг 6. Проверка

1. Найдите бота в Telegram → `/start`.
2. *Подобрать размер* → введите 168, 90, 70 → бот выдаст рекомендацию.

## Шаг 7. Заполнить каталог

В Google Таблице, лист `Каталог`, по одной строке на товар:

| Артикул | Название | Материал | Уход |
|---|---|---|---|
| 245678901 | Платье льняное белое | 100% лён | Стирка 30°, не отжимать, сушить расправленным |

Меняете таблицу — бот сразу видит изменения, перезапускать не нужно.

## Шаг 8. QR-код

1. На своём компьютере откройте `generate_qr.py`, замените `your_bot_username` на свой username бота (без @).
2. В терминале: `pip install qrcode[pil]` → `python generate_qr.py`.
3. Появится `qr-code.png`. Печатайте и вкладывайте в посылки.

## Если что-то не работает

| Симптом | Что проверить |
|---|---|
| Бот не отвечает | Railway → Logs → есть ли «Бот запущен»? Правильный ли `BOT_TOKEN`? |
| «Не нашла товар» | В листе `Каталог` есть строка с этим артикулом? |
| Странный размер | Проверьте `SIZE_CHART` в `config.py` — подстройте под свой бренд |
